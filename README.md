# Redrob Candidate Finder

Ranks ~100k candidates against a job description using embeddings + rule-based signals. Outputs the top 100 best-fit candidates.

---

## How to Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Get the data
Place `candidates.jsonl` inside the `data/` folder. (Too large for GitHub — get it from the hackathon source.)

### 3. Precompute (run once — no time limit, may use the network)
```bash
python precompute/embedded_candidates.py   # text -> vectors (~33 min; raise BATCH_SIZE on a strong PC)
python precompute/build_features.py         # rule-based signals  (~1-2 min)
python precompute/flag_honeypots.py         # impossible-profile traps (seconds)
python precompute/setup_reranker.py         # downloads the cross-encoder into artifacts/reranker/ (one-time, online)
python precompute/build_rerank_texts.py     # compact per-candidate docs for the reranker (seconds)
```
These write everything to `artifacts/`. First run downloads the embedding model (~90MB) and the cross-encoder (~120MB) — needs internet **once**, here in precompute only.

> **Compute split:** precompute has no time limit and may use the network. The **ranking step (`rank.py`) must not** — it runs CPU-only, offline, in ≤5 min. That's why the models are downloaded here and saved into `artifacts/` so `rank.py` can load them locally with `HF_HUB_OFFLINE=1`.

### 4. Rank
```bash
python rank.py
```
Done. Results are in `submission.csv` (~40s, CPU-only, no network).

Optional — see how much the cross-encoder changed the ranking vs the rule/embedding score alone:
```bash
python compare_rankings.py
```

## How It Works (quick)

1. **Embed** — converts the JD and every candidate's profile into vectors using `all-MiniLM-L6-v2` (bi-encoder, for fast recall over 100k)
2. **Features** — rule-based signals: experience fit, location, notice, engagement, and **evidence-weighted** skill matching (verified hits vs keyword stuffing)
3. **Honeypots** — flags candidates with impossible dates/math (planted traps)
4. **Rank** — a **3-part score**: additive *relevance core* (cosine + verified skills + experience + product-company) × *bounded soft modifiers* (location / notice / engagement) × *hard gates* (zero out honeypots & irrelevant titles, penalize genuine mismatches)
5. **Rerank** — the top-400 shortlist is reordered by a local **cross-encoder** (`ms-marco-MiniLM-L-12-v2`) that reads the JD and candidate jointly, then blended 50/50 with the Phase-1 score → top 100. Runs CPU-only, offline, in ~40s.
