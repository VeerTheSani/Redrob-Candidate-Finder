# Redrob Candidate Finder

Ranks ~100k candidates against a job description using embeddings + rule-based signals. Outputs the top 100 best-fit candidates.

---

## How to Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Get the data
Place `candidates.jsonl` inside the `data/` folder. (Too large for GitHub — get it from the hackathon source.)

### 3. Precompute (run once — no time limit)
```bash
python precompute/embedded_candidates.py
python precompute/build_features.py
python precompute/flag_honeypots.py
```
These write everything to `artifacts/`. First run downloads the embedding model (~90MB, needs internet once).

### 4. Rank
```bash
python rank.py
```
Done. Results are in `submission.csv`.


---

## How It Works (quick)

1. **Embed** — converts the JD and every candidate's profile into AI vectors using `all-MiniLM-L6-v2`
2. **Features** — extracts rule-based signals: experience fit, location, skills, disqualifiers
3. **Honeypots** — flags candidates with impossible dates/math (planted traps in the dataset)
4. **Rank** — multiplies cosine similarity × all feature scores, penalizes disqualifiers, zeros out honeypots → top 100
